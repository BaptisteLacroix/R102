Les différents formats d'images :  Matriciels, vectoriels et avec ou sans perte.

Légende :  du chien

  1. Une image matricielle, ou carte de points (de l'anglais bitmap), est une image constituée d'une matrice de points colorés. C'est-à-dire, constituée d'un tableau, d'une grille, où chaque case possède une couleur qui lui est propre et est considérée comme un point. Il s'agit donc d'une juxtaposition de points de couleurs formant, dans leur ensemble, une image.

    Wikipedia

  2. Une image vectorielle (ou image en mode trait), en informatique, est une image numérique composée d'objets géométriques individuels, des primitives géométriques (segments de droite, arcs de cercle, courbes de Bézier, polygones, etc.), définis chacun par différents attributs (forme, position, couleur, remplissage, visibilité, etc.) et auxquels on peut appliquer différentes transformations (homothéties, similitude, rotations, écrasement, mise à l'échelle, extrusion, inclinaison, effet miroir, dégradé de formes, morphage, symétrie, translation, interpolation, coniques ou bien les formes de révolution). Elle se différencie en cela des images matricielles (ou images bitmap), qui sont constituées de pixels.

    Wikipedia

  3. La compression d'image est une application de la compression de données sur des images numériques. Cette compression a pour utilité de réduire la redondance des données d'une image afin de pouvoir l'emmagasiner sans occuper beaucoup d'espace ou la transmettre rapidement. La compression d'image peut être effectuée avec perte de données ou sans perte. La compression sans perte est souvent préférée là où la netteté des traits est primordiale :  schémas, dessins techniques, icônes, bandes dessinées. La compression avec perte, plus radicale, est utile pour les transmissions à bas débit, mais dégrade la qualité de l'image restituée. Les méthodes de compression sans perte sont également préférées là où la précision est vitale :  balayages médicaux ou numérisations d'images pour archivage. Les méthodes avec perte restent acceptables pour des photos dans les applications où une perte mineure de fidélité (parfois imperceptible) est tolérée pour réduire les coûts de stockage ou d'envoi.

    Wikipedia

Quelle sont les différences entre les formats PNG, GIF, JPG et SVG ? 

  1. Le format PNG ( Portable Network Graphics) est un format ouvert d'images numériques. Il a été créé initialement pour remplacer le format GIF (Graphics Interchange Format), qui lui était à l'époque propriétaire et dont la compression était brevetée.

    Wikipedia

  2. Le format GIF a été mis au point en 1987 par le groupe CompuServe, dirigé par l'informaticien Steve Wilhite (en), pour permettre le téléchargement d'images en couleur. Ce format utilise l'algorithme de compression sans perte LZW, nettement plus efficace que l'algorithme RLE utilisé par la plupart des formats alors disponibles (PCX, ILBM puis BMP).

    Wikipedia

  3. Le format JPG ou JPEG est un bon choix pour les images compressées avec pertes. C'est le format le plus populaire et parmi les mieux pris en charge. PNG sera préférable si on préfère une meilleure fidélité à l'image et WebP/AVIF permettront d'avoir de meilleurs taux de compression.

    MDN web docs

  4. Le format SVG est un format d'image vectorielle basé sur XML pour les graphiques bidimensionnels avec prise en charge de l'interactivité et de l'animation. La spécification SVG est un standard ouvert développé par le World Wide Web Consortium (W3C) depuis 1999. C'est un format d'image vectoriel idéal pour les éléments d'interface utilisateur, les icônes ou diagrammes qui nécessitent de s'afficher précisément quelle que soit l'échelle.

    MDN web docs / Wikipedia

Pour quel usage est conseillé chaque format ?

  • Il est conseillé d'utiliser le format png pour les photos et les visuels riches en couleur,
    le format gif pour les images animées colorées,
    le format png pour les graphiques, icônes et logos simples
    et le format svg pour les images vectorisées, afin de ne pas perdre de qualité lors d'un changement de format.

De nouveaux formats d'images ont vu le jour récemment :  leurs points forts et leurs points faibles.

    1. Le nouveau format JPEG2000 est une nouvelle norme de compression d'images avec ou sans perte de données. Son point faible est qu'il est incompatible avec le format JPEG. JPEG2000 était breveté par la société texane Forgent Networks jusqu'en 2006, mais le brevet a expiré. Aujourd'hui, le format n'est pas breveté.

    2. Le format WebP est un format d'image matricielle développé et mis à disposition du public par Google. C'est un excellent choix pour les images statiques ou animées. Les taux de compression de ce format sont meilleurs que pour PNG ou JPEG. Ce format prend également en charge de meilleures profondeurs pour les couleurs, les frames animées, la transparence, etc. On l'utilise dans le but de conserver une bonne qualité d'image tout en réduisant leur poids de 25%. Son point faible est que ce format est déconseillé pour l'archivage. Il est disponible sur différents supports (Linux, Windows 10, MacOs, iOS et Android)

    3. Le format APNG Bon choix pour les animations sans perte de qualité. AVIF et WebP ont de meilleures performances mais une moins bonne couverture des navigateurs. L’APNG a besoin d’être optimisé.

    4. Le format JPEG-XR a été développé par Windows et sert à améliorer la précision des couleurs JPG pour avoir une meilleure compression. Son point faible est qu'il contient des pertes de détails fins lors de la compression. Ce format n'est supporté par tous les navigateurs web.

    5. Le format BPG est un format basé sur la compression avec ou sans perte. Le but de ce format est de remplacer le format JPEG grâce à des meilleurs taux de compression et grâce à sa qualité d'image. Bien qu'il n'y ait pas de support natif intégré pour BPG dans les navigateurs grand public, les sites Web peuvent toujours fournir des images BPG à tous les navigateurs en incluant une bibliothèque JavaScript écrite par Bellard Son point faible est que la compression est très lente.

    6. Le format FLIF est un format d'image sans perte qui prétend surpasser PNG, WebP sans perte, BPG sans perte et JPEG sans perte 2000 en termes de taux de compression sur une variété d'entrées. il est destiné à être libre, sans perte et gratuit. Ce format n'a pas de brevet mais ne peut pas encore être utilisé sur différents navigateurs. FLIF n'est plus en cours de développement.

    7. Le format JPEGXL est un format de fichier graphique libre de droits qui prend en charge la compression avec et sans perte. Il est conçu pour surpasser les formats raster existants et ainsi devenir leur remplacement universel. Il devient de plus en plus compatible avec les supports d'aujourd'hui, il est encore en développement.

    8. Le format AVIF est fomat de fichier graphique spécialisé dans le stockage d'images ou de séquences compréssé avec AV1 dans le HEIF format conteneur. Il est en compétition avec HEIC qui utilise le même stockage de format. La version 1 de AVIF a été finalisé en Février 2019. Ce format est libre de droits mais son point faible est qu'il est encore en développement. C'est un format avec de bonnes performances et sans licence commerciale qui peut être utilisé pour les images statiques comme animées. Il offre une meilleure compression que PNG et JPEG et une meilleure profondeur de couleur. Il est préférable d'utiliser des formats alternatifs via l'élément <picture> pour s'adapter aux différentes prises en charge des navigateurs.

    MDN web docs /  Wikipedia

En Général vaut-il mieux utiliser "_blank" ou "_self" comme valeur de target (et pourquoi  ? )

  • Il vaut mieux utiliser "_blank" car cela permet d'accéder à une autre page web sans perdre celle actuelle, cela évitera de surcharger l'onglet actuel et il est plus simple de revenir sur l'ancienne page si celle si s'ouvre dans un autre onglet.
